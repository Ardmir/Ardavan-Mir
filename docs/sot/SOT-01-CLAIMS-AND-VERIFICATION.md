## Claim status labels

- `[CONFIRMED]` — safe to use based on user confirmation, source files, public profile, or visible repo/site evidence.
- `[INFERRED]` — strong framing based on confirmed facts; verify before using as a precise factual claim.
- `[VERIFY]` — needs confirmation before publication.
- `[DO NOT USE]` — remove or replace unless verified.

## Verification workflow

When new content is proposed:

1. Identify every factual claim.
2. Tag each claim as `[CONFIRMED]`, `[INFERRED]`, `[VERIFY]`, or `[DO NOT USE]`.
3. Move verified claims into this file.
4. Update `SOT-08-CHANGELOG.md` with the verification event.
5. Do not publish résumé or portfolio copy containing unresolved `[VERIFY]` placeholders.
